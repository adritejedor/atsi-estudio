# Release Checklist

## Autorización

- [ ] Versión aprobada.
- [ ] Rama correcta.
- [ ] Despliegue autorizado.
- [ ] Entorno confirmado.

## Código

- [ ] Working tree entendido.
- [ ] Commit identificado.
- [ ] `npm ci`.
- [ ] `npm run validate`.
- [ ] Build de producción.
- [ ] Tests relevantes.
- [ ] Sin warnings críticos.

## Configuración

- [ ] Variables de entorno.
- [ ] Secretos.
- [ ] URLs de producción.
- [ ] Analytics.
- [ ] Consentimiento.
- [ ] Firebase/proveedor.
- [ ] Dominio.
- [ ] SSL.
- [ ] Backup cuando aplica.

## Contenido

- [ ] Datos de contacto.
- [ ] Precios.
- [ ] Legal.
- [ ] Sitemap.
- [ ] Robots.
- [ ] Metadata.
- [ ] Imágenes.

## Despliegue

- [ ] Proyecto/target correcto.
- [ ] Comando registrado.
- [ ] Resultado correcto.
- [ ] No hubo operación destructiva inesperada.

## Smoke test

- [ ] Home.
- [ ] Rutas directas.
- [ ] Navegación.
- [ ] Formulario.
- [ ] Integraciones.
- [ ] Cookies.
- [ ] Analytics.
- [ ] 404.
- [ ] Dominio raíz.
- [ ] `www`.
- [ ] Móvil.
- [ ] Consola.

## Cierre

- [ ] Informe de release.
- [ ] Cliente/responsable informado.
- [ ] Rollback disponible.
- [ ] Incidencias registradas.
