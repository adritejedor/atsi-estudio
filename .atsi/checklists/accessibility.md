# Accessibility Checklist

## Estructura

- [ ] `lang` correcto.
- [ ] Landmarks.
- [ ] Un `main`.
- [ ] Headings lógicos.
- [ ] Listas semánticas.
- [ ] Botones y enlaces correctos.

## Teclado

- [ ] Todos los controles accesibles.
- [ ] Orden lógico.
- [ ] Sin trampas.
- [ ] Escape funciona en overlays.
- [ ] Menú móvil usable.
- [ ] Skip link cuando aporta valor.

## Focus

- [ ] Visible.
- [ ] Contraste.
- [ ] No se pierde.
- [ ] Se restaura tras diálogos.

## Formularios

- [ ] Labels visibles.
- [ ] Errores asociados.
- [ ] Required comunicado.
- [ ] Autocomplete.
- [ ] No depende del placeholder.
- [ ] Estados anunciados.
- [ ] No depende solo del color.

## Imágenes e iconos

- [ ] Alt correcto.
- [ ] Decorativas con alt vacío.
- [ ] Iconos con nombre accesible.
- [ ] No hay imágenes de texto innecesarias.

## Color y visual

- [ ] Contraste AA.
- [ ] Focus visible.
- [ ] Links distinguibles.
- [ ] Estados no dependen solo del color.
- [ ] Zoom 200%.
- [ ] Reduced motion.

## Interacción

- [ ] Touch targets razonables.
- [ ] Sin hover obligatorio.
- [ ] No hay contenido que parpadee.
- [ ] Carruseles o movimiento controlables.

## Herramientas

- [ ] Auditoría automática.
- [ ] Revisión manual de teclado.
- [ ] Flujos críticos revisados.
