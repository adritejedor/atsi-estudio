# ATSIestudio — Optimize SEO

Este procedimiento revisa SEO técnico, semántico y de contenido sin convertir el texto en una acumulación artificial de keywords.

---

## 1. Comprender intención

Para cada ruta:

- Keyword principal.
- Intención.
- Público.
- Conversión.
- Tema principal.
- Relación con otras páginas.

Una URL no debe intentar posicionar de forma principal para demasiadas intenciones distintas.

---

## 2. Revisar arquitectura

Comprobar:

- Rutas claras.
- Jerarquía.
- Navegación.
- Enlaces internos.
- Breadcrumbs cuando sean útiles.
- Ausencia de páginas huérfanas.
- Evitar duplicación de intención.

---

## 3. Metadatos

Para cada ruta pública:

- Title único.
- Description única.
- Canonical.
- Open Graph.
- Twitter Card si aplica.
- Idioma.
- Robots.

Evitar títulos genéricos y descripciones repetidas.

---

## 4. Contenido

Comprobar:

- Un `h1` claro.
- Headings lógicos.
- Respuesta a la intención.
- Copy útil.
- Keywords naturales.
- Servicios y ubicación explicados.
- CTA alineada.
- Sin texto de relleno.
- Sin promesas no demostrables.

No forzar densidad de keywords.

---

## 5. SEO local

Cuando corresponda:

- Nombre comercial coherente.
- Área geográfica.
- Datos de contacto consistentes.
- Contenido local real.
- Datos estructurados válidos.
- Enlaces a perfiles empresariales relevantes.
- Páginas locales solo si tienen contenido único.

No crear decenas de páginas locales casi idénticas.

---

## 6. Imágenes

- Nombre descriptivo.
- Alt útil.
- Dimensiones.
- Formato.
- Compresión.
- Contexto textual.
- No usar alt con keywords repetidas.

---

## 7. Indexación

Comprobar:

- `robots.txt`.
- `sitemap.xml`.
- Canonical.
- Rutas privadas no indexadas.
- Errores 404.
- Redirecciones.
- SPA fallback.
- Estado HTTP cuando la infraestructura lo permita.

---

## 8. Datos estructurados

Usar solo esquemas aplicables y con datos visibles.

Ejemplos:

- Organization.
- LocalBusiness.
- Service.
- BreadcrumbList.
- FAQPage cuando el contenido cumpla requisitos.

Validar el JSON-LD.

No inventar ratings ni reseñas.

---

## 9. Rendimiento y accesibilidad

SEO también depende de:

- Velocidad.
- Mobile.
- Semántica.
- Contenido accesible.
- Navegación.
- Core Web Vitals.

Coordinar con estándares de rendimiento y accesibilidad.

---

## 10. Analítica

Cuando exista:

- Definir conversiones.
- Respetar consentimiento.
- Evitar duplicar eventos.
- Verificar rutas SPA.
- No exponer datos personales.

---

## 11. Validación

- Revisar HTML generado.
- Validar metadatos.
- Validar schema.
- Comprobar sitemap.
- Comprobar robots.
- Revisar enlaces.
- Ejecutar Lighthouse SEO.
- Ejecutar `npm run validate`.

---

## 12. Informe

Registrar por ruta:

- Keyword principal.
- Title.
- Description.
- H1.
- Canonical.
- Schema.
- Enlaces internos.
- Pendientes de contenido.
